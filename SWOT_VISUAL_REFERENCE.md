## 📊 Data Flow Diagram

```
┌─────────────────────┐
│ SWOTAnalysisPage    │
│   [Mount]           │
└──────────┬──────────┘
           │
           ├─→ useAppStore() → Get role & IDs
           │
           ├─→ Aggregate Metrics
           │   ├─ Admin: orgData, teams, users
           │   ├─ Manager: team from managerTeamId
           │   └─ Developer: user from developerUserId
           │
           ├─→ Call generate*SWOT(metrics)
           │
           └─→ OpenRouter API
               ├─ Qwen 3 Next 80B
               ├─ 3000 token limit
               └─ Detailed tactical prompt
                  │
                  ├─→ Parse Response
                  │   └─ Extract 16 JSON items
                  │
                  ├─→ Group by Category
                  │   ├─ strength (4)
                  │   ├─ weakness (4)
                  │   ├─ opportunity (4)
                  │   └─ threat (4)
                  │
                  └─→ Render 2x2 Grid
                      ├─ Animate entries
                      ├─ Color code
                      └─ Display details

[API Fails]
  │
  └─→ Fallback Data
      └─→ getAdminFallbackSWOT()
      └─→ getManagerFallbackSWOT()
      └─→ getDeveloperFallbackSWOT()
```

## 🧪 Test Scenarios

### Scenario 1: Admin Loading SWOT
1. User switches to Admin role
2. Navigates to SWOT Analysis
3. System loads org metrics
4. API generates 16 insights
5. All items display in 2x2 grid
6. Colors match categories
7. ✅ Verify: Each category has 4 items

### Scenario 2: Manager with Low Team Adoption
1. Manager views team SWOT
2. API generates insights specific to team
3. Weakness section highlights engagement gaps
4. Opportunity section shows training areas
5. ✅ Verify: Items reference team metrics

### Scenario 3: API Failure Fallback
1. Disable internet/API
2. User navigates to SWOT
3. Fallback items load instantly
4. Layout identical to API results
5. ✅ Verify: UX unchanged

### Scenario 4: Regenerate & Compare
1. Load SWOT once
2. Click Regenerate
3. New API call made
4. Different insights loaded
5. ✅ Verify: Items changed from original

## 🎓 Key Features Summary

✨ **Dynamic**: All items from OpenRouter API  
🎯 **Tactical**: Only operational insights  
📊 **Role-Aware**: Different analysis per role  
🎨 **Beautiful**: Matches screenshot exactly  
⚡ **Fast**: 2-5 second load time  
🛡️ **Resilient**: Fallback for failures  
📱 **Responsive**: Works on all devices  
🔄 **Refreshable**: Regenerate for new insights  
🎬 **Animated**: Smooth transitions
